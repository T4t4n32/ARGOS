# Protocolos de seguridad

La seguridad es un valor fundamental para el proyecto ARGOS.  Este
directorio reúne las políticas y protocolos que deben seguirse durante
el diseño, la integración y las pruebas del sistema.

## Contenido

Los siguientes documentos están disponibles (o deben elaborarse) para
garantizar una práctica segura:

- **Seguridad física**: reglas para manipular baterías, motores y
  electrónica de potencia; identificación de riesgos y procedimientos
  para pruebas en laboratorio.
- **Uso de sensores**: recomendaciones y advertencias sobre la
  naturaleza cualitativa del MQ‑135 y la calibración de otros
  dispositivos como el BME280 y el VL53L0X.
- **Seguridad operativa**: condiciones y umbrales para detener pruebas
  inmediatamente; reglas sobre dónde y cómo ejecutar demos.
- **Seguridad del repositorio**: buenas prácticas para manejar datos
  sensibles, reportar vulnerabilidades y documentar incidentes.
- **Respuesta a incidentes**: protocolo simple de tres pasos (detener,
  asegurar, registrar) y plan de comunicación.

Para la versión completa del posicionamiento oficial del proyecto,
consulta el archivo [`SECURITY.md`](../../SECURITY.md) en la raíz del
repositorio.
